Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I will recommend the client to make sure they use programs to sanitization of data, this will be for example use of the Sanitize-html package in NodeJS or other programs to sanitizing the SQL. Other recommendations will also get the client to research the use of authorization and programs for authentication. As always recommend using well-known third party applications and research these before implementing. If something seems to be to good to be true is free, there will probably be a reason. I would recommend the client to have cyber security in mind in all the parts of the website, from start to finish. This is a demanding task, but if everyone involved in the making of the website has security in mind the process of making the website more secure will be easier. Not only to secure the actual website, but also to protect its users. However, this is a demanding task and require a lot of knowledge about cyber security. If the client does not have the knowledge nor the funds to hire own security responsible employee in the organization, I would recommend they use money on outside resources to help to keep the website and its users secure. Using money on security seems to a lot of clients to be an unnecessary use of funds, but the potentially loss of customers, funds, or the loss of reputation due to a security breech would damage the business so much more than the cost of external use of security.
